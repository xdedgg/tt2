服务工单系统融合流程与实施方案

版本：V1.0

日期：2025年1月7日

一、文档概述

本文档旨在阐述服务工单流程体系的整改方案，聚焦于BOSS工单与客服工单的深度融合。方案基于BOSS工单流，打通装维预开通、上门施工及竣工授权等关键环节，构建"营业、装维、客服"服务闭环。

文档针对当前业务中存在的工单数据不同步、装维"首响时长"缺失以及"有线电视及宽带安装及时率"不达标等问题，提供了技术整改路径。通过数据同步，实现关键节点的系统自动记录，确保服务指标的合规性。

同时，本文档规划了涵盖"营业厅办理、客服接单、网格员全流程"在内的三大服务场景，明确了各岗位的预期变化与作业规范。内容涵盖功能建设需求、异常处理机制，为系统的后续开发、功能测试及人员培训提供参考。

二、预期变化

2.1 关键指标问题及解决方案

问题现象（指标待完善） 解决方法
数据同步问题：
工单系统与BOSS系统新装工单数据不同步，装维工单的"首响时长"指标无系统统计数据或数据无效。 		打通系统，实现数据实时双向同步。
时效指标不达标：
1. 有线电视安装首次响应时长
2. 宽带业务安装首次响应时长
3. 有线电视安装及时率
4. 宽带安装及时率 		系统自动记录关键节点（接单T1、预开通T2、派单T3、上门T4、竣工T4、回访T5……）等时间戳，替代人工填单。基于这些准确、连续的时序数据，预设规则自动或手动计算各项指标。

2.2 对分公司营业员的影响

  • 系统自动记录所有操作时间，无需手动填写
  • 新增派单、回单操作入口
  • 支持一键查看关联工单，提升处理效率
  • 客户安装情况可溯源

2.3 对网格员的影响

  • 支持智慧宝端全流程操作
  • 新增施工拍照功能，自动获取工单类型
  • 系统记录上门时间和服务时长

2.4 对客服的影响

  • 自动关联BOSS工单数据，形成完整服务闭环
  • 所有工单（包括BOSS工单和无需上门的工单）均汇总，方便今后生成报表进行服务质量分析
  • 需对现场照片审查及服务流程监察，以确保功能上线效果

三、功能建设与开发需求

  1. 已具备BOSS系统的工单流，支持装维预开通，上门施工竣工后再开通授权。
  2. 需增加智慧宝端BOSS工单派单和回单功能，流程进度与BOSS同步。支持网格员通过移动设备完成全流程操作，实现移动端工单派发、接收、处理、回单，并确保移动端与PC端数据实时同步。
  3. 需与客服系统打通，包括BOSS工单与客服单关联，以及将BOSS工单数据汇总到客服系统。建立双向数据通道，实现BOSS工单与客服工单自动关联，保障数据一致性，并以最优方案进行同步，保障业务开通时效性。
  4. 需在现有的客户及受理单页面增加查看对应工单的功能，达到溯源效果。实现业务全过程可追溯，支持关联工单查看及历史工单查询功能。
  5. 需在智慧宝端增加工单施工、竣工时的拍照等功能。支持现场照片上传、存储和查看，支持多张照片上传，拍照记录与工单关联，智慧宝与BOSS同步互通。同时考虑是否支持添加无需BOSS操作的工单类型（拍照记录服务过程）。
  6. 需制定工作要求方案，异常处理机制，进行多场景测试，BOSS中以工单流方式配置开户、增机等营业项目。
  7. 需各个岗位人员配置和管理要求。

四、流程总览

服务工单流程包含6个核心节点，形成一个完整的业务闭环。流程节点如下所示：

1
接单
客服通过电话接收客户服务请求，创建客服工单并派发给相关执行人员。
2
预开通
初步处理客户需求，收费，在BOSS中录入产品，产品状态为未激活。
3
BOSS派单
将BOSS工单派发给网格员。
4
装维，竣工
网格员上门服务、拍照记录、BOSS回单（自动触发预开通的受理单竣工）。
5
汇总数据
在此之前以最优方案完成数据关联及同步至客服系统。
6
结束流程
回访确认、满意度记录、服务流程闭环。

五、三大服务场景

系统支持三种主要服务场景，覆盖不同业务入口和流程路径：

A场景：营业厅办理

场景描述：用户到营业厅办理业务，网格员不进行BOSS业务操作。

适用流程节点：节点2 → 节点3 → 节点4 → 节点5 → 节点6（跳过节点1）

操作人员：营业员（PC端操作）、网格员（仅施工）

流程特点：

  • 营业员通过PC端收钱和业务受理操作
  • 营业员通过PC派BOSS工单给网格员
  • 网格员仅负责现场施工和拍照
  • 也适合网格员不操作收钱和BOSS开户的所有场景
B场景：客服电话接单

场景描述：客服电话接单并派客服单给网格员。

适用流程节点：节点1 → 节点2 → 节点3 → 节点4 → 节点5 → 节点6（完整流程）

操作人员：客服、网格员

流程特点：

  • 客服派客服单给网格员
  • 网格员全流程操作，包括施工、拍照、回单
  • 客服工单与BOSS工单需关联
C场景：网格员全流程

场景描述：网格员自己联系的客户并进行全流程操作。

适用流程节点：节点2 → 节点3 → 节点4 → 节点5 → 节点6（跳过节点1）

操作人员：网格员

流程特点：

  • 网格员全流程独立操作
  • 网格员通过智慧宝派BOSS单给自己
  • 无客服参与，直接从预开通开始

六、异常处理与流程中断场景

中断位置 异常情况 处理方式
节点1、2之间 结束客服单，无需创建BOSS工单 直接结束客服工单，不触发后续BOSS流程
节点2、3之间 营业厅销户处理 结束工单，客户通过其他途径重新下单则重新受理
节点3、4之间 网格员无法上门：退单、改派 将工单退回或改派给其他网格员
客户反悔或跑单 营业厅销户处理，结束工单。如客户通过其他途径下单则重新受理

重要业务规则

  • 客服接单后客户跳单：被派单的网格员结束工单。客户从其余途径重新开始记录
  • 已受理状态客户反悔：进行BOSS销户，由网格员或营业厅人员结束工单（考虑到销户操作只有营业厅能做，所以结束工单可选择由谁来操作）
  • BOSS工单必须回单：所有BOSS工单必须完成回单操作，数据传给客服系统，结束，闭环
  • 既有客服工单也有BOSS工单：B场景中的网格员接到客服工单后，业务受理时自动创建BOSS工单，工单竣工时，与对应的客服工单关联，可自动结束客服工单或增加弹窗提示：“检测到存在未关闭的客服工单[单号]，是否一并结束？尽量减少网格员的回单工作量。
  • 无需BOSS操作的工单：如需对协助用户操作，或咨询投诉类服务进行拍照记录，则需要系统支持网格员创建无需BOSS操作的工单类型
